# Superpowers — Planning & Design Artifacts

> **Status: design & planning material, not current product documentation.**
>
> Everything under this directory is a *proposal* or *work-in-progress*
> captured during a planning sprint. The documents here describe features
> that **may or may not be implemented** — they are kept in the repo so
> that the rationale behind in-flight work is reviewable, but they should
> not be read as a description of Watchdog's current behaviour.
>
> If you're looking for user-facing documentation, start at:
>
> - [`docs/README.md`](../README.md) — entry point into the product docs
> - [`docs/architecture-guide.md`](../architecture-guide.md) — what the code does today
> - [`docs/roadmap.md`](../roadmap.md) — prioritised list of planned work
>
> For the current status of each spec below, see the top of each file — the
> header has been updated to state whether the feature is shipped, partially
> shipped, or still a proposal.

## Directories

- `plans/` — step-by-step implementation plans; one-time operator guides.
- `specs/` — design specifications; the "why" and "what" behind each plan.

## Provenance

Files are named `YYYY-MM-DD-<slug>.md` after the date the proposal was
written. Dates do **not** imply a shipping order.
